# Tinkoff Portfolio (Google Apps Script)

Google Sheets-инструмент для учёта инвестиционного портфеля через **Tinkoff Invest API v2**.

Поддерживает:
- облигации;
- фонды;
- акции;
- опционы;
- построение `Dashboard`;
- автообновление по триггерам;
- экспорт диаграмм и отправку отчёта в Telegram.

---

## Текущий статус

На текущий момент в проекте уже работают:
- отдельные листы `Bonds`, `Funds`, `Shares`, `Options`;
- меню обновления для акций;
- включение акций в weekly/monthly pipeline;
- short + long карточка по акциям;
- загрузка по акциям **из текущего портфеля**, а не из `Input`;
- fallback-логика для `Dashboard`: если общего портфельного dashboard ещё нет, используется старый `buildBondsDashboard()`.

Это значит, что модуль акций уже можно использовать независимо от финального рефакторинга `Dashboard`.

---

## Возможности

### 1. Листы с данными
- `Bonds` — облигации: полное обновление и обновление только цен.
- `Funds` — фонды: полное обновление и обновление только цен.
- `Shares` — акции: полное обновление и обновление только цен.
- `Options` — опционы: полное обновление и обновление только цен.

### 2. Акции (`Shares`)
Лист `Shares` заполняется **только по текущим позициям в портфеле**.

В листе доступны:
- идентификация бумаги: `FIGI`, тикер, название;
- позиционные метрики: количество, средняя цена, текущая цена;
- расчётные поля: инвестировано, рыночная стоимость, `P/L (руб)`, `P/L (%)`;
- карточка инструмента: сектор, страна риска, валюта, биржа, тип акции, доступность шорта, блокировка;
- фундаментальные метрики: капитализация, выручка, `EBITDA`, чистая прибыль, `EPS`, `P/E`, `P/S`, `P/B`, `EV/EBITDA`, `Debt/Equity`, `NetDebt/EBITDA`, `Free Float`, `Beta`, `Shares Outstanding`, `52w High`, `52w Low`, `ROE`, `ROA`, `ROIC`.

### 3. Dashboard
- текущий `Dashboard` строится через `buildBondsDashboard()`;
- в меню и scheduler уже предусмотрен переход на общий `buildPortfolioDashboard()`, если он добавлен в проект;
- экспорт диаграмм и текстовой сводки в Telegram поддерживается через `dashboard_export.gs`.

### 4. Telegram
- отправка приветственного сообщения;
- отправка PNG-диаграмм с `Dashboard`;
- отправка текстовой статистики по секциям dashboard.

### 5. Планировщик
- еженедельное автообновление;
- ежемесячное автообновление;
- акции включены в общий pipeline наряду с облигациями, фондами и опционами.

## Логика источников данных

### `Input`
Лист `Input` остаётся общей входной таблицей идентификаторов:
- **A** — `FIGI` облигаций;
- **B** — `FIGI` фондов;
- **C** — `FIGI` акций;
- **D** — `UID` опционов (`uid:xxxxxxxx-...`).

### Важно по акциям
Несмотря на наличие колонки `C` в `Input`, текущий модуль `Shares` работает **не из `Input`**, а **из фактических текущих позиций портфеля**.

То есть:
- `Input!C` может использоваться как вспомогательная колонка/заготовка;
- для `Shares` источником истины является портфель пользователя через API.

## Структура скриптов

### Базовые файлы
- `main_menu.gs` — меню в Google Sheets.
- `core.gs` — кэш, служебные утилиты, рендер/форматирование.
- `utils.gs` — конвертеры `Quotation`, `MoneyValue`, времени и вспомогательные функции.
- `tinkoff_api.gs` — обёртки Tinkoff API (`Users`, `Instruments`, `MarketData`, `Operations`).
- `input_loader.gs` — загрузка идентификаторов в `Input`.
- `tgbot_api.gs` — Telegram API.

### Обновление листов
- `update_bonds.gs` — обновление листа `Bonds`.
- `update_funds.gs` — обновление листа `Funds`.
- `update_shares.gs` — обновление листа `Shares`.
- `update_options.gs` — обновление листа `Options`.

### Dashboard / экспорт
- `dashboard_bonds.gs` — текущий построитель `Dashboard` по облигациям.
- `dashboard_export.gs` — экспорт диаграмм/секций и отправка в Telegram.
- `dashboard_shares_helpers.gs` — чистые helper-функции по листу `Shares` для будущих equity/portfolio блоков.
- `dashboard_portfolio.gs` — общий портфельный dashboard (если добавлен в проект).

### Планировщик
- `sheduler.gs` — weekly/monthly trigger jobs и общий pipeline.

> Примечание: файл в проекте сейчас называется именно `sheduler.gs`.

## Быстрый старт

1. Создайте Google Sheets → **Расширения → Apps Script**.
2. Добавьте файлы проекта в Apps Script.
3. Задайте токен:
   - через меню: **Тинькофф • Портфель → Настройки → Задать/сменить токен**;
   - либо через свойства скрипта.
4. Проверьте доступ:
   - **Портфель → Проверка доступа**;
   - **Портфель → Список аккаунтов**.
5. Загрузите `Input`, если используете облигации/фонды/опционы:
   - **Данные → Загрузить FIGI в Input (все типы)**.
6. Обновите листы:
   - **Облигации → Полное обновление**;
   - **Фонды → Полное обновление**;
   - **Акции → Полное обновление**;
   - **Опционы → Полное обновление**.
7. Постройте dashboard:
   - **Сводка → Обновить Dashboard**.

## Проверка модуля акций

### Smoke-test
1. Откройте меню **Акции**.
2. Запустите **Полное обновление**.
3. Убедитесь, что появился лист `Shares`.
4. Проверьте, что заполнились:
   - `FIGI`, тикер, название;
   - количество, средняя цена, текущая цена;
   - `Инвестировано`, `Рыночная стоимость`, `P/L (руб)`, `P/L (%)`;
   - фундаментальные поля хотя бы для части бумаг.

### Проверка обновления только цен
1. Запустите **Акции → Обновить только цены**.
2. Убедитесь, что:
   - меняются `Текущая цена` и `Время цены`;
   - пересчитываются `Рыночная стоимость`, `P/L (руб)`, `P/L (%)`.

## Автообновление

### Еженедельный триггер
Пример установки weekly-refresh на понедельник, 9:00:

```js
installWeeklyAutoRefresh(9)
```

### Ежемесячный триггер
Пример установки monthly-refresh на 1-е число месяца, 8:00:

```js
installMonthlyAutoRefresh(1, 8)
```

### Что делает pipeline
`weeklyRefreshAndNotify()` и `monthlyAutoRefreshJob()` по очереди вызывают:
- `updateBondsFull()`
- `updateFundsFull()`
- `updateSharesFull()`
- `updateOptionsFull()`
- `buildPortfolioDashboard()` или fallback на `buildBondsDashboard()`

## Telegram-экспорт

Основные функции:

## Известные особенности

1. Не у всех акций фундаментальные поля будут заполнены полностью. Это зависит от доступности данных в API.
2. `Shares` работает по текущим позициям портфеля, а не по списку `Input!C`.
3. До финального рефакторинга общий `Dashboard` может оставаться bond-centric, даже если лист `Shares` уже полностью работает.
4. Telegram summary чувствителен к layout новых блоков dashboard, поэтому dashboard/export лучше обновлять согласованно.
